1. **Q1a**  
VERDICT: AGREE. The identity is correct: for a linear row operator \(H_M(U)=A_M U + b_M\), the difference is \(H_M(\tilde U)-H_M(U)=A_M(\tilde U-U)\), and any additive source term cancels. Same-side stencil legs have \(\tilde U=U^n\), so only cross-interface legs contribute. This remains true for the RSG elastic update because the velocity and stress equations are linear combinations of old state components with fixed coefficients. Algorithm recommendation: assemble the correction rows directly from the same bulk stencil weights, using `U* - U^n` only on cross-Γ legs. This matches the LP04 read-substitution view in Lombard & Piraux 2004, DOI [10.1016/j.jcp.2003.09.024](https://www.sciencedirect.com/science/article/pii/S0021999103005126).

2. **Q1b**  
VERDICT: AGREE. Bulk-then-add is algebraically identical to a read-time LP04 operator as long as the correction is computed from the old time slice \(U^n\), not from the post-bulk \(U^{n+1}\). A non-irregular target row has no cross-Γ stencil legs by definition, so LP04 and bulk are identical for that row even if one of its source cells is itself irregular for its own update. If a target cell \(N\) reads a source cell \(M\) on the opposite side, then \(N\)'s stencil crosses Γ, so \(N\) is an irregular row and gets its own \(\Delta_N\). The source cell being irregular is not the criterion; the target row’s read geometry is. Algorithm recommendation: correct all and only target rows whose stencil support crosses Γ, on both sides of the interface, then scatter-add into `U.forward` at those target rows.

QUESTION 1 — Correction-term decomposition (math)
═══════════════════════════════════════════════════════════════════════

LP04 Eq 43-44 says: at irregular cell M, the discrete operator H reads U^n at SAME-side stencil legs and U* at CROSS-Γ legs.

For a LINEAR discrete operator H (which our RSG is — it's a sum of stencil-weight × U value products with no nonlinear function applied to U):

    U^(n+1, LP04)_M = H_M(Ũ_neighbours)
    U^(n+1, bulk)_M  = H_M(U^n_neighbours)
    Δ_M := U^(n+1, LP04)_M - U^(n+1, bulk)_M
         = H_M(Ũ - U^n)
         = sum over CROSS-Γ legs (α,β) at M:
             stencil_weight[M, (α,β), c_out, c_in]
             × (U*_(M+α,β, c_in) - U^n_(M+α,β, c_in))

where Ũ has U^n at same-side legs (zero difference, no contribution) and U* at cross-Γ legs.

Each cell M has typically 5-15 CROSS-Γ legs (half the SO=8 diagonal stencil). Total non-zeros in the sparse correction matrix C:
  n_cells × n_state² × n_cross_leg_avg ≈ 12000 × 25 × 10 = ~3M

Per-timestep flow:
  1. Standard Devito op.apply(time_m=n, time_M=n, dt=dt) advances U → U^(n+1, bulk) everywhere.
  2. SpMV #1: U_star_per_cell = N @ flatten(U^n)  [precomputed N row vectors per β.3].
  3. SpMV #2: delta_per_cell = C @ (U_star - U^n_at_cross_neighbours).
  4. Apply: U.data[c, target_i[cell], target_j[cell]] += delta_per_cell[cell, c] at all irregular cells.

# Devito - Finite Difference DSL for Symbolic PDE Computation

## Overview

Devito is a Python-based domain-specific language (DSL) and code generation framework for solving partial differential equations (PDEs) through symbolic computation. You write the math; Devito generates optimized C/C++ loops with automatic parallelization (OpenMP, MPI, GPU).

## CRITICAL: Use Devito DSL, Not NumPy

**Always generate Devito DSL code, not raw NumPy implementations.**

The Devito example notebooks (especially in `examples/cfd/`) follow a pedagogical pattern:
1. First, they show a **NumPy reference implementation** (manual loops, array slicing)
2. Then, they show the **Devito DSL implementation** (symbolic, auto-optimized)

When writing code with this skill, **always use the Devito DSL approach**:

```python
# BAD - NumPy manual loops (DO NOT generate this)
for n in range(nt):
    un = u.copy()
    u[1:-1] = un[1:-1] + nu * dt / dx**2 * (un[2:] - 2*un[1:-1] + un[:-2])

# GOOD - Devito DSL (ALWAYS generate this)
from devito import Grid, TimeFunction, Eq, Operator
grid = Grid(shape=(nx,), extent=(lx,))
u = TimeFunction(name="u", grid=grid, time_order=1, space_order=2)
eq = Eq(u.forward, u + nu * dt * u.dx2)
Operator(eq)(time=nt, dt=dt)
```

**Why Devito DSL:**
- Automatic C code generation with compiler optimizations
- Automatic parallelization (OpenMP, MPI, GPU)
- Symbolic derivatives (no manual stencil coding)
- Dimension-agnostic code (same code works 1D/2D/3D)

## CRITICAL: Update Equations Must Have Explicit LHS Assignment

**Devito Operators require equations with an explicit assignable LHS (left-hand side).**

The Operator compiles equations into C code. Each equation must tell Devito **what variable to update**. A symbolic PDE constraint like `Eq(u.dt2, c**2 * u.dx2)` does NOT work because it has no assignable target.

```python
# WRONG - Symbolic PDE form passed directly to Operator
# This generates INVALID C code and fails to compile!
pde = Eq(u.dt2, c**2 * u.dx2)
op = Operator([pde])  # ERROR: "expression is not assignable"

# CORRECT - Use solve() to derive update equation
from devito import solve
pde = Eq(u.dt2, c**2 * u.dx2)
update = Eq(u.forward, solve(pde, u.forward))  # Explicit LHS: u.forward
op = Operator([update])  # Works correctly

# ALSO CORRECT - Manual stencil with explicit LHS
update = Eq(u.forward, 2*u - u.backward + dt**2 * c**2 * u.dx2)
op = Operator([update])  # Works correctly
```

**The Rule:** Every `Eq()` passed to `Operator()` must have a **single assignable symbol** on the LHS:
- `u.forward` - next time step
- `u[t+1, x, y]` - indexed access
- `p` - static function update

**Never pass** `Eq(u.dt2, ...)` or `Eq(u.dt, ...)` directly to an Operator.

## CRITICAL: Use Proper Unit Scaling for Numerical Stability

**Always scale quantities to O(1) to prevent floating-point catastrophe, especially for FP32/GPU.**

Finite difference stencils are "catastrophic cancellation machines"—every derivative computation subtracts nearby values. When field values are large (e.g., pressure in Pascals ~10^6), the meaningful differences lose significant digits.

### The Principle: Scale to O(1)

This applies to **all domains**, not just seismic modeling. Make sensible unit choices based on your problem:

| Domain | Sensible Choice | Poor Choice | Reasoning |
|--------|----------------|-------------|-----------|
| Seismic | km/s, km, g/cm³ | m/s, m, kg/m³ | Wave speeds 1-6 km/s become O(1) |
| CFD | Normalize L=1, U=1, use Re | SI units directly | Reynolds number formulation |
| Heat transfer | Normalize to ΔT=1, L=1 | Kelvin, meters | Dimensionless form |
| Microfluidics | μm, μs | m, s | Characteristic scales are microscale |

**The goal:** All coefficients in your discretized equations should be O(1), not 10^6 or 10^-6.

```python
# GOOD - Quantities scaled to O(1) for the problem at hand
# Seismic example: km/s units
grid = Grid(shape=(201, 201), extent=(2.0, 2.0))  # 2 km domain
c = 1.5  # km/s

# CFD example: normalized units
grid = Grid(shape=(100, 100), extent=(1.0, 1.0))  # Unit domain
Re = 100  # Reynolds number
nu = 1.0 / Re  # Dimensionless viscosity

# BAD - Large magnitude ratios cause precision loss
c = 1500  # m/s when domain is in km → mixed scales
pressure = 101325  # Pa when other quantities are O(1)
```

### Why This Matters

- **FP32 has ~7 decimal digits** - magnitude ratios > 10^6 cause absorption/cancellation
- **GPU computing uses FP32** - NVIDIA Tensor Cores: 125 TFLOP/s FP16 vs 7.8 TFLOP/s FP64
- **Long simulations accumulate error** - 10^6 timesteps leaves only ~4 reliable digits in FP32

**Rule of thumb:** Keep all quantities within 10^-3 to 10^3 for FP32 safety. Choose units that make this natural for your problem.

See `references/units-numerical-precision.md` for detailed guidance on non-dimensionalization, unit verification with **pint** (recommended for development), and common scaling errors.

## CRITICAL: Set `space_order` on All Functions Used Inside Derivatives

**When a `Function` appears inside an expression that is subsequently differentiated, it MUST have `space_order` matching the intended FD accuracy.** Otherwise Devito silently uses a low-order, asymmetric stencil that produces wrong results or numerical instability.

```python
# BAD — C defaults to space_order=1, corrupts the composed derivative
C = Function(name='C', grid=grid)  # space_order=1 by default!
u = TimeFunction(name='u', grid=grid, space_order=8)
rhs = (C * u.dx).dx  # Uses fd_order=1 for outer .dx — UNSTABLE

# GOOD — C has matching space_order
C = Function(name='C', grid=grid, space_order=8)
u = TimeFunction(name='u', grid=grid, space_order=8)
rhs = (C * u.dx).dx  # Uses fd_order=8 — correct
```

This affects **any composed derivative** pattern: `(C * u.dx).dx`, `(C * u.dy).dy`, divergence of stress from Hooke's law, etc. It does NOT affect `u.dx2` (direct second derivative) or expressions where the Function is only multiplied, not differentiated through (e.g., `damp * update_expr`).

**Diagnostic:** Print `expr.fd_order` — if it shows `(1,)` when you expect `(8,)`, this is the bug.

See `references/faq-troubleshooting.md` → "Composed Derivatives Inherit `fd_order` from Lowest `space_order`" for full details.

## Core Workflow

```
1. Grid        → Define discrete domain (shape, extent)
2. Fields      → Create unknowns (TimeFunction) and coefficients (Function)
3. PDE         → Write derivatives symbolically (.dx, .dx2, .laplace, .dt, .dt2)
4. Update      → Explicit step: Eq(u.forward, ...) or solve(pde, u.forward)
5. Compile/Run → Operator([eqs...])(time=Nt, dt=dt)
6. BCs         → Extra Eq(...) constraints or subdomains
7. Extract     → Final solution at u.data[Nt % (time_order+1)] (rolling buffer)
```

For complete API reference (Grid, Function, TimeFunction, SparseTimeFunction, Eq, solve, Operator, derivatives, vector/tensor fields), see `references/dsl-semantics.md`.

def normal_and_tangent(dip_deg: float) -> tuple[np.ndarray, np.ndarray]:
    """Unit normal (n) and tangent (t) to the dipping line.

    Convention matches signed_distance_dipping_line:
        n = (sin θ, cos θ)  pointing into the upper (fluid) layer
        t = (cos θ, -sin θ) along the line, perpendicular to n

    For LP04's (x', y') parametrisation, (x', y') = t.
    """
    th = np.radians(dip_deg)
    n = np.array([np.sin(th), np.cos(th)])
    t = np.array([np.cos(th), -np.sin(th)])
    return n, t


def project_to_interface(point_xy: np.ndarray, dip_deg: float,
                         anchor_xy: np.ndarray) -> np.ndarray:
    """Orthogonal projection of `point_xy` onto the dipping line through
    `anchor_xy`. Returns the projected (x_P, y_P) coordinate."""
    n, _ = normal_and_tangent(dip_deg)
    d_signed = np.dot(point_xy - anchor_xy, n)
    return point_xy - d_signed * n


def gather_disk_neighbours(target_xy: np.ndarray,
                            P_xy: np.ndarray,
                            sdf: np.ndarray,
                            dx: float,
                            q_factor: float = 3.5,
                            shape: tuple[int, int] = None,
                            anchor_grid_xy: tuple[float, float] = (0.0, 0.0),
                            ) -> tuple[list[tuple[int, int]],
                                        list[tuple[int, int]]]:
    """Gather grid points within the disk B of radius q_factor·dx around P.

    Returns:
        B_1 : list of (i, j) grid indices on the fluid side (sdf > 0)
        B_2 : list of (i, j) grid indices on the solid side (sdf < 0)

    The point on Γ itself (sdf ≈ 0) is skipped.
    """
    Nx, Ny = sdf.shape
    radius = q_factor * dx
    # Grid coords corresponding to sdf[i, j]:
    #   x_i = anchor_grid_xy[0] + i * dx; y_j = anchor_grid_xy[1] + j * dx
    # Find candidate index range around P:
    P_i = (P_xy[0] - anchor_grid_xy[0]) / dx
    P_j = (P_xy[1] - anchor_grid_xy[1]) / dx
    i_lo = max(0, int(np.floor(P_i - q_factor)))
    i_hi = min(Nx, int(np.ceil(P_i + q_factor)) + 1)
    j_lo = max(0, int(np.floor(P_j - q_factor)))
    j_hi = min(Ny, int(np.ceil(P_j + q_factor)) + 1)

    B_1, B_2 = [], []
    for i in range(i_lo, i_hi):
        x_i = anchor_grid_xy[0] + i * dx
        for j in range(j_lo, j_hi):
            y_j = anchor_grid_xy[1] + j * dx
            dist = np.hypot(x_i - P_xy[0], y_j - P_xy[1])
            if dist > radius:
                continue
            s = sdf[i, j]
            if s > 1e-12 * dx:
                B_1.append((i, j))
            elif s < -1e-12 * dx:
                B_2.append((i, j))
            # cells essentially ON the interface: skip
    return B_1, B_2

def C1_zero_numeric(t: np.ndarray) -> np.ndarray:
    """LP04 Eq 11: fluid-side 2×3 jump matrix at order 0, evaluated for
    a specific tangent t = (x', y'). Couples [v·n] = 0 and pressure."""
    xp, yp = float(t[0]), float(t[1])
    return np.array([
        [-yp, xp, 0],
        [0, 0, xp**2 + yp**2],
    ], dtype=np.float64)


def C2_zero_numeric(t: np.ndarray) -> np.ndarray:
    """LP04 Eq 11: solid-side 2×5 jump matrix at order 0.

    State ordering U_2 = (v_1, v_2, σ_11, σ_12, σ_22)^T.
    Row 1: normal velocity component.
    Row 2: normal-stress continuity n^T σ n · |t|^2 = -p · |t|^2.
    """
    xp, yp = float(t[0]), float(t[1])
    return np.array([
        [-yp, xp, 0, 0, 0],
        [0, 0, yp**2, -2 * xp * yp, xp**2],
    ], dtype=np.float64)


def L1_zero_numeric(t: np.ndarray) -> np.ndarray:
    """LP04 Eq 11: fluid-side 1×3 boundary-condition row (empty for
    perfect fluid — pressure is unconstrained beyond the jump). The
    paper carries this row as zero so that K_1^0 = G_1^0 reduces
    cleanly via the SVD."""
    return np.zeros((1, 3), dtype=np.float64)


def L2_zero_numeric(t: np.ndarray) -> np.ndarray:
    """LP04 Eq 11: solid-side 1×5 row — vanishing tangential shear
    t^T σ n = 0."""
    xp, yp = float(t[0]), float(t[1])
    return np.array([[0, 0, xp * yp, yp**2 - xp**2, -xp * yp]],
                     dtype=np.float64)


def G_fluid_zero() -> np.ndarray:
    """LP04 Appendix A.1 at k=0: 3×3 identity (V_1^0 = U_1^0)."""
    return np.eye(3, dtype=np.float64)


def G_solid_zero() -> np.ndarray:
    """LP04 Appendix A.2 at k=0: 5×5 identity (V_2^0 = U_2^0)."""
    return np.eye(5, dtype=np.float64)


def build_K_via_svd(L: np.ndarray, G: np.ndarray) -> np.ndarray:
    """Per LP04 Eq 26: V_i^k = K_i^k · W_i^k where K_i^k spans the
    nullspace of L_i^k · G_i^k.

    Returns K such that (L · G) · K ≈ 0; columns of K form an
    orthonormal nullspace basis.
    """
    LG = L @ G
    # Nullspace via SVD: smallest singular vectors with sigma ≈ 0
    U, s, Vt = svd(LG, full_matrices=True)
    # Tolerance for "zero" singular values:
    tol = max(LG.shape) * np.finfo(LG.dtype).eps * (s.max() if s.size else 0.0)
    rank = int((s > tol).sum())
    K = Vt[rank:, :].T  # nullspace columns
    return K

@dataclass
class PackedESIMTables:
    """Fixed-stride, Devito-loadable ESIM tables.

    Padding convention:
      - cell_B_ij[c, b_idx, :] = (-1, -1) for b_idx >= cell_n_B[c]
      - cell_N[c, :, b_idx, :] = zeros((5, 5)) for b_idx >= cell_n_B[c]
      - Pad-zero in cell_N is mathematically equivalent to "no
        contribution from this B-slot" since the dot product
        N · (U^n)_B picks up that B-slot's contribution multiplied
        by 0.

    Applying U*_{cell c} = sum over b_idx of cell_N[c, :, b_idx, :]
    contracted against the gathered (U^n)_B at the (i,j) given by
    cell_B_ij[c, b_idx, :]. Out-of-bounds (-1) entries should be
    masked (and they ARE auto-masked by the zero-coefficient padding).
    """
    n_cells: int
    n_B_max: int
    state_components: int
    cell_target_ij: np.ndarray   # (n_cells, 2) int32
    cell_side: np.ndarray         # (n_cells,) int8
    cell_n_B: np.ndarray          # (n_cells,) int32
    cell_B_ij: np.ndarray         # (n_cells, n_B_max, 2) int32
    cell_N: np.ndarray            # (n_cells, sc, n_B_max, sc) float64
    cell_cond_M: np.ndarray       # (n_cells,) float64
    k: int

    @property
    def memory_mb(self) -> float:
        total = (self.cell_N.nbytes + self.cell_B_ij.nbytes
                 + self.cell_target_ij.nbytes + self.cell_side.nbytes
                 + self.cell_n_B.nbytes + self.cell_cond_M.nbytes)
        return total / 1e6


def pack_esim_tables(tables: ESIMTables,
                       n_B_max: int | None = None,
                       ) -> PackedESIMTables:
    """Convert variable-length `ESIMTables` to padded fixed-stride
    `PackedESIMTables`. If n_B_max is None, use the actual max from
    the input tables."""
    if n_B_max is None:
        n_B_max = max(len(b) for b in tables.B_ij)
    sc = tables.state_components
    n_cells = tables.n_cells

    cell_target_ij = tables.target_ij.copy()
    cell_side = tables.cell_side.copy()
    cell_n_B = np.array([len(b) for b in tables.B_ij], dtype=np.int32)
    cell_B_ij = np.full((n_cells, n_B_max, 2), -1, dtype=np.int32)
    cell_N = np.zeros((n_cells, sc, n_B_max, sc), dtype=np.float64)

    for c in range(n_cells):
        nB = len(tables.B_ij[c])
        cell_B_ij[c, :nB, :] = tables.B_ij[c]
        # Reshape N from (sc, sc * n_B) to (sc, n_B, sc):
        # N's column ordering matches gather_UB: (b_0_comp_0, b_0_comp_1,
        # ..., b_0_comp_4, b_1_comp_0, ..., b_(n_B-1)_comp_4).
        # So column index k = b_idx * sc + comp_in.
        N_reshaped = tables.N[c].reshape(sc, nB, sc)
        cell_N[c, :, :nB, :] = N_reshaped

    return PackedESIMTables(
        n_cells=n_cells,
        n_B_max=int(n_B_max),
        state_components=sc,
        cell_target_ij=cell_target_ij,
        cell_side=cell_side,
        cell_n_B=cell_n_B,
        cell_B_ij=cell_B_ij,
        cell_N=cell_N,
        cell_cond_M=tables.cond_M.copy(),
        k=tables.k,
    )


def apply_packed_python(packed: PackedESIMTables,
                          field: np.ndarray,
                          U_star: np.ndarray | None = None,
                          ) -> np.ndarray:
    """Apply the packed ESIM tables via a host-side Python loop.

    Equivalent to `apply_esim_python` from precompute, but operating on
    the padded fixed-stride layout. Used to verify the packing is
    lossless before β.4.
    """
    if U_star is None:
        U_star = np.zeros_like(field)
    sc = packed.state_components
    for c in range(packed.n_cells):
        i, j = packed.cell_target_ij[c]
        nB = packed.cell_n_B[c]
        # Gather (U^n)_B in cell_N's expected order
        UB = np.empty(sc * nB)
        for b_idx in range(nB):
            ib, jb = packed.cell_B_ij[c, b_idx]
            UB[b_idx * sc : (b_idx + 1) * sc] = field[:, ib, jb]
        # N_flat shape (sc, sc * nB): keep the unpadded slice
        N_flat = packed.cell_N[c, :, :nB, :].reshape(sc, sc * nB)
        U_star[:, i, j] = N_flat @ UB
    return U_star


def apply_packed_einsum(packed: PackedESIMTables,
                         field: np.ndarray,
                         U_star: np.ndarray | None = None,
                         ) -> np.ndarray:
    """Vectorised apply using NumPy fancy-indexing + einsum.

    Faster than the Python loop variant. Used as a reference for the
    eventual Devito-compiled equivalent in β.4 — also demonstrates the
    sparse-gather pattern Devito will need to express symbolically.
    """
    if U_star is None:
        U_star = np.zeros_like(field)
    sc = packed.state_components
    n_cells = packed.n_cells
    n_B_max = packed.n_B_max

    # Fancy-index gather: B_field[c, b, comp] = field[comp, cell_B_ij[c, b, 0],
    #                                                   cell_B_ij[c, b, 1]]
    # Replace -1 entries with index 0 for the gather; we'll mask via
    # cell_N padding (zeros at unused b-slots).
    safe_B_ij = np.where(packed.cell_B_ij == -1, 0, packed.cell_B_ij)
    bi = safe_B_ij[:, :, 0]  # (n_cells, n_B_max)
    bj = safe_B_ij[:, :, 1]
    # B_field shape: (sc, n_cells, n_B_max). Use np.take over the
    # spatial dims via advanced indexing:
    B_field_per_comp = np.empty((sc, n_cells, n_B_max), dtype=field.dtype)
    for comp in range(sc):
        B_field_per_comp[comp] = field[comp, bi, bj]
    # Transpose to (n_cells, n_B_max, sc) so einsum aligns:
    B_field = B_field_per_comp.transpose(1, 2, 0)
    # Mask invalid slots: cell_N has zeros there, so the contribution
    # is zero regardless of B_field's value. No explicit mask needed.

    # U*[c, comp_out] = sum over (b_idx, comp_in) of
    #     cell_N[c, comp_out, b_idx, comp_in] * B_field[c, b_idx, comp_in]
    U_star_per_cell = np.einsum('cobi,cbi->co', packed.cell_N, B_field)
    # Scatter into U_star
    ci = packed.cell_target_ij[:, 0]
    cj = packed.cell_target_ij[:, 1]
    for comp in range(sc):
        U_star[comp, ci, cj] = U_star_per_cell[:, comp]
    return U_star

    diff_einsum = np.max(np.abs(U_star_unpacked - U_star_einsum))
    print(f"\n  Apply timings:")
    print(f"    apply_esim_python (unpacked):    {t_unpack:.3f}s")

def _pi_scalar_at_offset(dxo: float, dyo: float, k: int) -> np.ndarray:
    """Recursive Taylor weights (3^k entries) at offset (dxo, dyo)."""
    if k == 0:
        return np.array([1.0])
    weights = [1.0]
    for _ in range(k):
        new_weights = []
        for w in weights:
            new_weights.append(w)
            new_weights.append(w * dxo)
            new_weights.append(w * dyo)
        weights = new_weights
    return np.array(weights)


def _pi_block_matrix(pi_scalar: np.ndarray, state_components: int
                      ) -> np.ndarray:
    """Tensor Π scalar weights with identity_{state_components} so that
    Π_block · U_k_vector = U^0 at the offset point (component-by-component).
    """
    n_mono = len(pi_scalar)
    M = np.zeros((state_components, state_components * n_mono))
    for m_idx, w in enumerate(pi_scalar):
        for c in range(state_components):
            M[c, m_idx * state_components + c] = w
    return M

    S_2_pinv = pinv(S_2)
    R_S2 = Vt_s2[rank_s2:, :].T  # nullspace columns of S_2 — shape (n_w2, n_lambda)
    n_lambda = R_S2.shape[1]

    # Per Eq 31: W_2 = S_2^{-1} [S_1 | R_{S_2}] (W_1, Λ)^T
    # i.e. W_2 = S_2_pinv @ S_1 @ W_1 + R_S2 @ Λ
    # (Note: LP04 writes [S_1 | R_S2] (W_1, Λ)^T as a single block
    #  acting on the concatenated (W_1, Λ). The natural decomposition
    #  is the linear sum above when |Λ| = nullspace dim.)
    # Build the W_2 = A_W1 · W_1 + A_lambda · Λ form:
    A_W1 = S_2_pinv @ S_1            # (n_w2, n_w1)
    A_lambda = R_S2                   # (n_w2, n_lambda)

    #   U(x_i, y_j) ≈ Π_{i,j}^0 · G_1 · K_1 · W_1 + O(Δx)
    # At k=0 the Π row vector is just scalar 1 (no derivative
    # contributions). So a row on B_1 reads:

    # i.e. M's B_1 block has rows = G_1 · K_1 with each grid point
    # contributing 5 rows (one per state component).
    #
    # For B_2:
    #   U(x_i, y_j) = G_2 · K_2 · W_2
    #              = G_2 · K_2 · (A_W1 · W_1 + A_lambda · Λ)
    M_block_b1_W1 = G_1 @ K_1                          # (5, n_w1)
    M_block_b1_lambda = np.zeros((5, n_lambda))
    M_block_b2_W1 = G_2 @ K_2 @ A_W1                   # (5, n_w1)
    M_block_b2_lambda = G_2 @ K_2 @ A_lambda           # (5, n_lambda)

    # Stack rows: |B_1|*5 + |B_2|*5 equations, unknown vector has
    # n_w1 + n_lambda = 5 + 1 = 6 components.
    n_b1, n_b2 = len(B1), len(B2)
    n_unknown = n_w1 + n_lambda
    n_eq = (n_b1 + n_b2) * state_components
    M = np.zeros((n_eq, n_unknown))
    row = 0
    for _ in B1:
        M[row:row + state_components, :n_w1] = M_block_b1_W1
        M[row:row + state_components, n_w1:] = M_block_b1_lambda
        row += state_components
    for _ in B2:
        M[row:row + state_components, :n_w1] = M_block_b2_W1
        M[row:row + state_components, n_w1:] = M_block_b2_lambda
        row += state_components

    info = {
        'G_1': G_1, 'G_2': G_2,
        'K_1': K_1, 'K_2': K_2,
        'S_1': S_1, 'S_2': S_2, 'S_2_pinv': S_2_pinv, 'R_S2': R_S2,
        'A_W1': A_W1, 'A_lambda': A_lambda,
        'n_w1': n_w1, 'n_w2': n_w2, 'n_lambda': n_lambda,
        'state_components': state_components,
    }

def assemble_N_k0_solid_side(B1: list[tuple[int, int]],
                              B2: list[tuple[int, int]],
                              sdf: np.ndarray,
                              dx: float,
                              anchor_grid_xy: tuple[float, float],
                              P_xy: np.ndarray,
                              t: np.ndarray,
                              state_components: int = 5,
                              ) -> tuple[np.ndarray, float, dict]:
    """For an irregular cell on the SOLID side (sdf < 0): compute the
    master row vector N such that U*_{I,J} ≈ N · (U^n)_B reproduces
    the fluid-side state (extended across Γ) at the cell.

    LP04 Eq 42: U*_{I,J} = Π_{I,J}^0 · G_1^0 · K_1^0 · M̄^{-1} · (U^n)_B
    At k=0 with Π = scalar 1: U*_{I,J} = G_1 · K_1 · W_1.

    Returns:
        N : (state_components, n_eq) row-vector matrix; one row per
            target state component.
        cond_M : condition number of M.
        info : matrix construction metadata.
    """
    M, info = build_M_matrix_k0(B1, B2, sdf, dx, anchor_grid_xy, P_xy, t,
                                  state_components)
    # M̄^{-1}: restrict M's pseudoinverse to the first n_w1 rows
    # (W_1 components only — drop Λ rows per Eq 40).
    M_pinv = pinv(M)
    Mbar_inv = M_pinv[:info['n_w1'], :]  # (n_w1, n_eq)
    # Π_{I,J}^0 = scalar 1 at k=0, so the U* mapping is:
    # U*_{I,J} = G_1 · K_1 · W_1 = G_1 · K_1 · Mbar_inv · (U^n)_B
    G_1 = info['G_1']
    K_1 = info['K_1']
    N = G_1 @ K_1 @ Mbar_inv  # (state_components, n_eq)
    cond_M = cond(M)
    return N, cond_M, info

@dataclass
class DevitoESIMApply:
    """Production Devito-compiled ESIM apply.

    Constructed once at setup from a PackedESIMTables; the resulting
    Operator can be reused across timesteps (just rebind U field data
    via the apply() method).
    """
    grid: object                   # Devito Grid
    n_cells: int
    n_offsets: int
    state_components: int
    offset_alphabet: np.ndarray    # (n_offsets, 2) int32 — (di, dj)
    op: object                     # Devito Operator
    # Devito Function handles (so the caller can interrogate / re-load)
    ti: object
    tj: object
    N_coeffs: object
    U_funcs: list                  # one Function per state component
    U_star: object                 # (cell_id, c_out) result Function

    def apply(self, field: np.ndarray) -> np.ndarray:
        """Apply ESIM gather to `field` (shape (5, Nx, Ny)).

        Returns U_star_sub data of shape (n_cells, 5) — same layout
        and ordering as PackedESIMTables.cell_target_ij.
        """
        for c, uf in enumerate(self.U_funcs):
            uf.data[:] = field[c]
        self.U_star.data[:] = 0.0
        self.op.apply()
        return self.U_star.data.copy()


def _compute_offset_alphabet(packed: PackedESIMTables
                              ) -> tuple[np.ndarray, np.ndarray]:
    """Compute the union of relative offsets across all cells and the
    per-cell mapping from slot b_idx to alphabet off_idx.

    Returns:
        offset_alphabet : (n_offsets, 2) int32 — (di, dj) per off_idx
        slot_to_off_idx : (n_cells, n_B_max) int32 — alphabet index
            for each cell's slot; -1 for unused slots (n_B[c] <= b_idx)
    """
    n_cells = packed.n_cells
    n_B_max = packed.n_B_max

    # Per-cell relative offsets
    # cell_B_ij shape: (n_cells, n_B_max, 2); target_ij shape (n_cells, 2)
    rel_offsets = packed.cell_B_ij - packed.cell_target_ij[:, np.newaxis, :]

    # Mask unused slots (B_ij == -1)
    valid_mask = packed.cell_B_ij[:, :, 0] != -1  # shape (n_cells, n_B_max)

    # Collect distinct (di, dj) over valid entries
    valid_offsets = rel_offsets[valid_mask]                # (M, 2)
    # Unique with stable ordering by (di, dj)
    offset_set = set(map(tuple, valid_offsets.tolist()))
    sorted_offsets = sorted(offset_set, key=lambda d: (d[0], d[1]))
    offset_alphabet = np.array(sorted_offsets, dtype=np.int32)
    offset_lookup = {tuple(o.tolist()): k for k, o in enumerate(offset_alphabet)}

    # Build per-cell slot→alphabet map
    slot_to_off_idx = np.full((n_cells, n_B_max), -1, dtype=np.int32)
    for c in range(n_cells):
        for b in range(n_B_max):
            if not valid_mask[c, b]:
                continue
            key = (int(rel_offsets[c, b, 0]), int(rel_offsets[c, b, 1]))
            slot_to_off_idx[c, b] = offset_lookup[key]

    return offset_alphabet, slot_to_off_idx


def _repack_coefficients(packed: PackedESIMTables,
                          offset_alphabet: np.ndarray,
                          slot_to_off_idx: np.ndarray) -> np.ndarray:
    """Repack cell_N from (cell_id, c_out, b_idx, c_in) to
    (cell_id, c_out, c_in, off_idx in alphabet), zeros elsewhere.
    """
    n_cells = packed.n_cells
    n_offsets = len(offset_alphabet)
    sc = packed.state_components
    N_repacked = np.zeros((n_cells, sc, sc, n_offsets), dtype=np.float64)
    for c in range(n_cells):
        nB = packed.cell_n_B[c]
        for b in range(nB):
            off_idx = slot_to_off_idx[c, b]
            if off_idx < 0:
                continue
            # cell_N[c, c_out, b, c_in] → N_repacked[c, c_out, c_in, off_idx]
            N_repacked[c, :, :, off_idx] = packed.cell_N[c, :, b, :]
    return N_repacked

def signed_distance_dipping_line(X, Y, dip_deg, anchor_x, anchor_y):
    """Signed distance from grid points (X, Y) to the dipping interface.

    Interface line: y - y_interface(x) = 0
    where y_interface(x) = anchor_y - tan(dip)·(x - anchor_x)
    (per ``tti_params.dipping_layer_mask`` convention; positive dip
    descends to the right).

    The line equation in normal form:
        tan(θ)·x + y - (anchor_y + tan(θ)·anchor_x) = 0
    Normal vector (unnormalised): (tan(θ), 1) — pointing UP (into
    upper layer, increasing y).
    Unit normal: (sin(θ), cos(θ)).

    SDF = (sin(θ)·(X - anchor_x) + cos(θ)·(Y - anchor_y))
        = signed perpendicular distance, POSITIVE in upper (water) layer
          and NEGATIVE in lower (TTI) layer.
    """
    th = np.radians(dip_deg)
    return np.sin(th) * (X - anchor_x) + np.cos(th) * (Y - anchor_y)


def identify_irregular_cells(sdf, stencil_offsets_diag):
    """For each cell (i, j) and each diagonal-stencil offset (Δi, Δj),
    mark the cell as irregular if SDF(i, j) and SDF(i+Δi, j+Δj) have
    opposite signs.

    Returns:
        irregular_mask : (Nx, Ny) bool — True if ANY tap crosses interface
        crossing_count : (Nx, Ny) int  — number of crossing taps per cell
    """
    Nx, Ny = sdf.shape
    irregular_mask = np.zeros_like(sdf, dtype=bool)
    crossing_count = np.zeros_like(sdf, dtype=np.int32)
    centre_sign = np.sign(sdf)

    for di, dj in stencil_offsets_diag:
        if di == 0 and dj == 0:
            continue
        # Source cells: (i, j); shifted reads: (i+di, j+dj).
        # The shift array maps (i, j) → SDF[i+di, j+dj].
        i_src = slice(max(0, -di), Nx - max(0, di))
        j_src = slice(max(0, -dj), Ny - max(0, dj))
        i_dst = slice(max(0, di), Nx - max(0, -di))
        j_dst = slice(max(0, dj), Ny - max(0, -dj))
        tap_sign = np.sign(sdf[i_dst, j_dst])
        ctr_sign = centre_sign[i_src, j_src]
        crosses = (tap_sign * ctr_sign) < 0  # opposite signs
        irregular_mask[i_src, j_src] |= crosses
        crossing_count[i_src, j_src] += crosses.astype(np.int32)

    return irregular_mask, crossing_count


def rsg_diagonal_stencil_offsets(so):
    """RSG diagonal stencil sample offsets at given SO.

    The M2 RSG diagonal stencil at SO=8 uses 4 taps along each of the
    (1, 1) and (1, -1) diagonals, at offsets m ∈ {-4, ..., +4}\\{0}.
    Returns the union of (m, m) and (m, -m) offsets for m = 1..so//2.
    """
    half = so // 2
    offsets = set()
    for m in range(1, half + 1):
        offsets.add((m, m))
        offsets.add((-m, -m))
        offsets.add((m, -m))
        offsets.add((-m, m))
    return sorted(offsets)

**Q1a — Correction-term decomposition (math)**
- VERDICT: AGREE
- Justification: The finite-difference operator $H$ for the linear elastic wave equation on a rotated-staggered-grid is strictly a linear combination of field values from neighboring nodes. Because $H$ is a linear operator, the principle of superposition applies directly: $H(\tilde{U}) = H(U^n + (\tilde{U} - U^n)) = H(U^n) + H(\tilde{U} - U^n)$. Here, $\tilde{U}$ represents the modified field where cross-interface points are replaced by their LP04 extrapolated values $U^*$, and $U^n$ is the unmodified field. The term $H(U^n)$ is exactly the output of the standard bulk Devito operator. Therefore, the correction term $\Delta_M = H(\tilde{U} - U^n)$ reduces to a sparse sum over only the cross-interface stencil legs, perfectly capturing the required LP04 modification without altering the bulk evaluation.
- Recommendation: Implement the correction as a sparse matrix-vector product that computes $\Delta_M$ by linearly combining $(U^* - U^n)$ at cross-interface legs, using the exact stencil weights from the RSG operator.
- Citations: Lombard & Piraux (2004), DOI: 10.1016/j.jcp.2003.09.024.

**Q1b — Bulk operator + post-hoc correction equivalence**
- VERDICT: AGREE
- Justification: At any given cell $N$, the bulk operator always computes $H_N(U^n)$. If $N$ is a regular cell (its stencil does not cross the interface $\Gamma$), then by definition $\tilde{U} = U^n$ for all nodes in its stencil. Consequently, the correction term $\Delta_N = H_N(\tilde{U} - U^n) = 0$, meaning the bulk operator's result is already exact and requires no modification. If $N$ is an irregular cell (its stencil crosses $\Gamma$), it incorrectly reads the "wrong-side" physical values during the bulk update. The post-hoc addition of $\Delta_N$ exactly subtracts the contribution of these wrong-side $U^n$ values and adds the contribution of the LP04 extrapolated $U^*$ values. Because this correction is applied selectively and exclusively to all cells whose stencils cross $\Gamma$ (on both sides), the final state algebraically perfectly matches a hypothetical read-time LP04 substitution.
- Recommendation: Use a two-pass approach each timestep: run the Devito bulk operator unmodified over the entire grid, then add the precomputed $\Delta_M$ correction vectors exclusively to the memory locations of the irregular cells.

**Q2a — Numerical library choice for ill-conditioned pseudoinverse**
- VERDICT: AGREE
- Justification: For constructing the pseudo-inverse of the LP04 Taylor-fit matrix $M$, robustness against rank-deficiency at degenerate interface geometries is critical. Instead, truncated SVD provides a mathematically sharp cutoff that preserves accuracy in well-conditioned directions while safely discarding noise from degenerate cell configurations. Using LAPACK's divide-and-conquer SVD (`gelsd`) efficiently computes the least-squares solution while simultaneously returning the effective rank. This rank information allows you to programmatically detect and flag severely ill-conditioned cells (e.g., the 4 edge cells) rather than relying on brittle topological heuristics like minimum points-per-side.
