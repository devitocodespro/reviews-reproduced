Architectural design review — Tier 3 ESIM implementation for the M2 RSG (Saenger 2000 rotated staggered grid) 45-degree X-mode interface artifact.

CONTEXT
═══════════════════════════════════════════════════════════════════════

A Devito-based elastic-wave solver using Saenger 2000 RSG on a 2D Petrobras seismic configuration (water layer over TTI elastic, dipping interface at theta in {0, 5, 20, 30} deg) produces a visible 45-degree X-mode diffraction artifact concentrated at the dipping fluid-solid interface. The artifact appears as 4-petal lobes at the source-near-field and 4-5 ghost wavefronts at later times. hf_ratio at dip=30 deg is ~9.8e-3 (vs ~2.1e-3 for clean methods like SSG-Yee and Irakarama 2026 Dual-Pair).

Multi-week empirical bisection has confirmed the artifact is ARCHITECTURAL — generated by the colocated diagonal-stencil x discontinuous-interface coupling — and CANNOT be addressed by:

 - stencil truncation order (SO sweep {4,8,12,16} — invariant)
 - time-stepping coupling (CFL sweep {0.1,0.2,0.3,0.4} — DAI fp64-invariant)
 - effective-medium averaging at the interface (BS04 -6%, Moczo 2002 -7%, Kristek 2017 +36% worse, Jiang-Zhang 2021 +68% worse)
 - source-stencil coupling (OumZhang 8-point recipe wired; §28b stale)
 - SO bumping in M61 STO-RSG (carries identical artifact)

Falsification diagnostic ✓ CONFIRMED interface origin: localised Cerjan damping at the irregular-cell band (those whose RSG SO=8 diagonal stencil crosses the analytical interface) drops hf_ratio by 84-91% (Tier 1B α=0.05). But DAI_curl inflates from 2.98 to 15+ — naive absorption is direction-asymmetric.

Tier 2 (pair-mask zeroing of stencil legs that cross Γ, no jump-condition enforcement) is WORSE than baseline: hf_ratio increases 812% (8.97e-2), artifact_E increases 76000x. Pile-up reflections off the discontinuous zero-derivative band. Confirms: no shortcut exists; full jump-condition machinery is required.

LOMBARD-PIRAUX 2004 ESIM RECIPE (paper-faithful transcription complete)
═══════════════════════════════════════════════════════════════════════

DOI 10.1016/j.jcp.2003.09.024. Just bootstrapped at reviews-reproduced/doi_10.1016_j.jcp.2003.09.024/ with 21/21 paper-faithfulness pytest assertions. Master equation (Eq 42):

  U*_{I,J} = Pi_{I,J}^k . G_1^k . K_1^k . Mbar^-1 . (U^n)_B

where for an irregular point (I,J) in Omega_2 (across-interface from the fluid),
 - Pi_{I,J}^k : Taylor row vector at (x_I - x_P, y_J - y_P) of order k
 - G_1^k : compatibility matrix Eq 25 (fluid vorticity reduction)
 - K_1^k : SVD nullspace of L_1^k.G_1^k boundary-condition matrix
 - Mbar^-1 : restriction of M^-1 to the W_1^k rows; M is the over-determined Taylor-fit matrix (Eq 38) built by stacking Eq 35 rows on B_1 and Eq 37 rows on B_2
 - (U^n)_B : numerical values gathered from the disk B of radius q ~ 3.5 dx around the orthogonal projection P of (I,J) onto Γ

Time-stepping at irregular points (Eq 43-44): apply the same discrete operator H as at regular points, but substitute U*_{i+α,j+β} for stencil legs that cross Γ (different medium from (i,j)), and keep U^n_{i+α,j+β} for same-side legs.

DIPPING-LINE SIMPLIFICATION (Petrobras-specific):
 - Γ is a single straight line ⇒ n, t, α_1, α_2 are constant
 - SVDs of L_i^k G_i^k and S_i^k computed ONCE at setup
 - Mbar^-1 depends only on integer cell offset of B relative to P
 - Per-irregular-point precomputed row vector:
     N_{I,J} := Pi_{I,J}^k . G_1^k . K_1^k . Mbar^-1
   → per-step cost is one mat-vec U*_{I,J} = N_{I,J} . (U^n)_B
   → paper §3.9 reports ~1% overhead at 400x400

QUESTIONS — Tier 3 IMPLEMENTATION DESIGN
═══════════════════════════════════════════════════════════════════════

Q1. PRECOMPUTE STRATEGY. The paper's recipe precomputes (G_i^k, K_i^k, Mbar^-1) at setup time and stores the per-irregular-point row vectors N_{I,J} as a NumPy array on the host. At each Devito timestep, we (a) gather (U^n)_B values from the stress fields via halo reads, (b) compute U*_{I,J} = N · (U^n)_B as a host-side dot product, (c) inject U* values into the Devito stencil at the right positions for Eq 43-44 substitution. Is this the cleanest decomposition, or is there a more elegant Devito-native approach? Specifically: does it make sense to express the whole substitution rule symbolically within a Devito Operator (via SubDomain + conditional mask + custom stencil), avoiding the host roundtrip entirely?

Q2. STENCIL SUBSTITUTION MECHANICS. RSG's diagonal-stencil reads stress tensor values at (i±1, j±1) corners. For a given irregular point (I,J), Eq 43 requires: if Ω(I+α, J+β) ≠ Ω(I,J), substitute the modified value U*_{I+α, J+β} computed via Eq 42 (using a DIFFERENT P projection per (α,β) pair, since each stencil leg might have its own near-Γ projection). Does this mean we need O(stencil_legs × irregular_points) modified values per timestep, not just O(irregular_points)? How should the Devito stencil express this substitution — via masked Eq's with a per-leg branch, via halo injection of U* arrays before each timestep, or via SparseFunction-based injection?

Q3. ORDER-k CHOICE. Paper §3.10 establishes that the ESIM preserves order min(k, scheme_order) near the interface. RSG is SO=8 in our production. k=2 (the paper's default) gives an O(dx^3) modified-value extrapolation. For RSG SO=8, should we use k=8 (matching scheme order — costs more setup, larger Mbar) or accept k=2 (cheaper, formally drops near-interface accuracy to 3rd order but the rest of the domain stays 8th)? Cite literature precedent for "ESIM matching order" choices in production solvers if available.

Q4. RISKS + DIAGNOSTIC TEST. The paper claims ~1% runtime overhead, but that's for a simple acoustic/elastic Cartesian configuration. For RSG (multiple staggered fields, TTI Cij coupling, diagonal stencil topology), what's the highest-likelihood failure mode of a straightforward implementation? Examples to consider: ill-conditioned Mbar (radius q too small), accuracy degradation when stencil legs cross Γ at oblique angles relative to the grid, MPI halo-exchange interaction with the cross-interface stencil reads, fluid-solid component-mixing in (U^n)_B (paper §3.6 notes the right-hand side mixes 3-component fluid + 5-component solid values for fluid-solid configurations). Recommend the SINGLE cheapest diagnostic test we could run BEFORE committing to multi-day implementation to confirm the design works at all.
